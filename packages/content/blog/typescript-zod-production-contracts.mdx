# TypeScript + Zod: How I Build End-to-End Type-Safe Production Systems

**Philosophy quote I live by:** *“TypeScript + Zod end-to-end is non-negotiable for anything production. If your data contracts aren’t typed and validated at runtime, you’re building on sand.”*

I am **Tanishk Khare**, **freelance TypeScript developer India** **teams** **recruit** when they want **contracts** **that** **survive** **network** **boundaries**, **LLM** **outputs**, and **third-party** **webhooks**. This article is the **pattern** **library**.

---

## 1. Why TypeScript Alone Fails at Runtime

**TypeScript** **erases** **at** **compile** **time**. **HTTP** **JSON**, **WebSocket** **frames**, **Claude** **tool** **arguments**, **Stripe** **webhooks** **—** **all** **arrive** **as** **unknown** **data**. **Without** **runtime** **validation**, **you** **only** **have** **hope**.

---

## 2. Zod as the Runtime Layer

**Zod** **schemas** **encode** **the** **same** **invariants** **your** **types** **express**, **but** **they** **run** **on** **real** **payloads**:

```typescript
import { z } from "zod";

export const CreateLeadBody = z.object({
  name: z.string().min(1),
  email: z.string().email(),
  message: z.string().min(20),
});

export type CreateLeadBody = z.infer<typeof CreateLeadBody>;
```

**Parse** **at** **the** **edge** **—** **route** **handler** **or** **server** **action** **—** **before** **business** **logic** **touches** **data**.

---

## 3. API Contracts (Fastify / Express)

```typescript
app.post("/api/leads", async (req, reply) => {
  const body = CreateLeadBody.parse(req.body);
  // body is typed — safe to pass down
});
```

**Same** **schema** **reused** **in** **OpenAPI** **generation** **if** **you** **add** **that** **layer**.

---

## 4. LLM Tool I/O (xTanBot.ai Style)

**Tool** **inputs** **from** **Claude** **must** **parse** **before** **execution**; **tool** **outputs** **must** **parse** **before** **re-injection** **into** **the** **message** **list**. **That** **is** **how** **you** **keep** **agentic** **AI** **from** **corrupting** **state**.

```typescript
export const SearchVisaInfo = z.object({
  countryCode: z.string().length(2),
  visaType: z.enum(["tourist", "work", "student"]),
});
```

---

## 5. Environment Variables

```typescript
export const Env = z.object({
  DATABASE_URL: z.string().url(),
  REDIS_URL: z.string().url(),
  ANTHROPIC_API_KEY: z.string().min(1),
});

export const env = Env.parse(process.env);
```

**Fail** **fast** **on** **boot**, **not** **mid-request**.

---

## 6. Prisma + Zod (Optional Codegen)

Teams **often** **use** **zod-prisma** **or** **custom** **mappers** **for** **DTOs** **exposed** **to** **clients** **—** **never** **expose** **raw** **DB** **rows** **without** **a** **schema** **you** **control**.

---

## 7. Error Shape

**Centralise** **Zod** **errors** **into** **consistent** **HTTP** **422** **responses** **with** **field** **paths** **—** **your** **frontend** **can** **bind** **validation** **to** **forms**.

---

## 8. Testing

**Zod** **schemas** **are** **the** **contract** **tests** **assert** **against** **—** **given** **bad** **input**, **parse** **throws** **predictably**.

---

## 9. When Not to Over-Zod

**Hot** **loops** **with** **known-safe** **internal** **data** **structures** **—** **sometimes** **trust** **after** **first** **parse**. **Never** **trust** **external** **input**.
